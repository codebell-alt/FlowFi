## BACKEND IMPLEMENTADO

### Arquitectura

```
Solicitud HTTP
    ↓
CORS Middleware
    ↓
Auth Middleware (valida JWT con Supabase)
    ↓
Router + Controlador
    ↓
Supabase Client
    ↓
PostgreSQL (a través de Supabase)
    ↓
Response JSON
```

## CÓMO EJECUTAR LOCALMENTE

### Paso 1: Configuración Inicial

```bash
cd Backend
python -m venv venv

# Windows
venv\Scripts\activate

# macOS/Linux
source venv/bin/activate

pip install -r requirements.txt
```

### Paso 2: Obtener Credenciales de Supabase

1. Ve a https://supabase.com
2. Crea un nuevo proyecto (o usa uno existente)
3. Copia la URL y la clave pública desde Settings > API
4. Copia la service role key

### Paso 3: Configurar Variables de Entorno

```bash
cp .env.example .env
# Editar .env con tus credenciales
```

### Paso 4: Crear Tablas en Supabase

```
En Supabase Dashboard > SQL Editor > New Query
Copiar y ejecutar el script SQL completo (proporcionado en README.md)
```

### Paso 5: Ejecutar el Servidor

```bash
python run.py
```

**Servidor disponible en:** http://localhost:8000

### Paso 6: Probar Endpoints

- **Swagger UI:** http://localhost:8000/docs
- **ReDoc:** http://localhost:8000/redoc

## GUÍA PASO A PASO DE SUPABASE

### 1. Crear Proyecto
- Registrarse en https://supabase.com
- Crear nuevo proyecto
- Hacer backend = PostgreSQL en la nube

### 2. Obtener Credenciales
- Settings > API > URLs y Keys
- SUPABASE_URL (Project URL)
- SUPABASE_KEY (anon public)

### 3. Crear Tablas
- SQL Editor > New Query
- Pegar el script SQL completo del README
- Ejecutar

### 4. Habilitar RLS
- Authentication > Policies
- Las políticas vienen en el script SQL

### 5. Conectar a la API
- Backend automáticamente se conecta
- Frontend usa Supabase Auth

## AUTENTICACIÓN EXPLICADA

### Flujo Completo:

1. **Usuario se registra/login en frontend**
   - El frontend usa Supabase Auth
   - Supabase retorna JWT token

2. **Frontend envía token al backend**
   - Header: `Authorization: Bearer <JWT>`
   - Cada request incluye el token

3. **Backend valida el token**
   - Middleware verifica con Supabase
   - Si es válido, extrae user_id
   - Si no, retorna 401 Unauthorized

4. **Backend filtra datos por usuario**
   - Automáticamente ve solo sus datos
   - RLS en BD asegura esto

5. **Response retorna a frontend**
   - Datos filtrados por usuario
   - Consistencia total

---

## CARACTERÍSTICAS DE SEGURIDAD

- JWT tokens validados en cada request
- Row Level Security (RLS) en Supabase
- CORS configurado para el frontend
- Contraseñas nunca se manejan (Supabase lo hace)
- Inyección SQL imposible (Supabase + Pydantic)
- Validación de permisos (roles admin/user)
